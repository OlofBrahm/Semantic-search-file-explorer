# Semantic Search File Explorer

**Semantic Search File Explorer** is a .NET 8.0 console application that performs semantic (vector-based) search on documents within a filesystem directory. It leverages vector embeddings and the HNSW (Hierarchical Navigable Small World) index to efficiently find and retrieve text documents relevant to a user's query—even if those documents don't contain exact keyword matches.

## Features

- **Semantic Search**: Utilizes machine learning embeddings to understand the meaning of queries and documents, returning the most relevant results.
- **File System Indexing**: Recursively scans a user-specified directory, indexing a wide variety of text-based documents.
- **Supported File Types**: Out-of-the-box support for `.txt`, `.md`, `.log`, `.json`, `.cs`, `.py`, `.html`, `.xml`, `.docx`, and others.
- **Efficient Nearest Neighbor Search**: Uses a custom HNSW (Hierarchical Navigable Small World) index for fast approximate nearest neighbor searches.
- **Console Interface**: Simple interactive prompt for entering queries and viewing results.

## Getting Started

### Prerequisites

- [.NET 8.0 SDK](https://dotnet.microsoft.com/en-us/download/dotnet/8.0)
- Windows OS (uses AlphaFS for advanced filesystem access; may require adaptation for other OSes)
- (Optional) ONNX-compatible ML models for advanced embeddings

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/OlofBrahm/Semantic-search-file-explorer.git
   cd Semantic-search-file-explorer
   ```

2. **Restore dependencies:**
   ```bash
   dotnet restore
   ```

3. **Build the project:**
   ```bash
   dotnet build
   ```

4. **Run the application:**
   ```bash
   dotnet run --project SimiliVec-Explorer.csproj
   ```

   The default root directory to index is set to `C:\Users\olleb\Documents`. Update this in `VectorService.cs` if you wish to point to a different folder.

## Project Architecture

- `Program.cs`: Entry point. Sets up core services and starts a REPL for search queries.
- `Services/StartupService.cs`: Orchestrates full directory scanning and indexing.
- `Services/SemanticIndexerService.cs`: Handles filesystem traversal, reads file contents, generates embeddings, and builds the search index.
- `Services/VectorService.cs`: Provides API for adding documents and executing semantic search queries.
- `hnsw-index/`: Contains the custom HNSW index implementation for fast nearest neighbor search.
- `DocumentStore/`: Maps document IDs to file paths and metadata.

## Example Usage

After starting the program, enter a search query at the prompt (e.g., `error handling in services`), and the program retrieves the closest-matching documents with their file paths.

```
Add search query: 
vector database implementation
 ::::::: Found document with ID: 12 ::::::: Content: C:\Users\olleb\Documents\notes\vector-db.txt
 ::::::: Found document with ID: 7 ::::::: Content: C:\Users\olleb\Documents\projects\semantic_explorer\README.md
```

## Customization

- **Change Root Path:**  
  Modify the `_rootPath` field in `VectorService.cs` to point to your desired folder.
- **Adjust Supported File Types:**  
  Edit the `SearchableExtensions` set in `SemanticIndexerService.cs`.
